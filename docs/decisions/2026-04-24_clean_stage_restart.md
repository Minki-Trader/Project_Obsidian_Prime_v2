# Decision Memo

## 결정(Decision, 결정)

프로젝트를 깨끗한 단계 구조(clean stage structure, 깨끗한 단계 구조)로 다시 시작한다.

## 이유(Why, 이유)

이전 v2 흐름(flow, 흐름)은 방어 리뷰 문장(defensive review language, 방어 리뷰 문장)을 많이 쌓았고, 탐색(exploration, 탐색)이 운영 게이트(operating gate, 운영 게이트)에 눌리는 느낌을 만들었다.

이번 재시작(restart, 재시작)은 가치 있는 자산(valuable assets, 가치 있는 자산)을 보존하되, 오래된 단계 결론(stage conclusions, 단계 결론)을 현재 진실(current truth, 현재 진실)에서 제거한다.

## 보존(Preserved, 보존)

- `.agents/skills`
- `docs/contracts`
- `docs/context`의 개념 노트(concept notes, 개념 노트)
- `data`
- 재사용 foundation 도구(reusable foundation tools, 재사용 기반 도구)

## 현재 진실에서 제거(Removed From Current Truth, 현재 진실에서 제거)

- 오래된 Stage 00부터 Stage 07까지의 폴더
- 오래된 결정 체인(decision chain, 결정 체인)
- 오래된 Stage 06과 Stage 07 모델 연구 스크립트(model-study scripts, 모델 연구 스크립트)
- 오래된 `Tier B(티어 B)` 이중 판정(dual-verdict, 이중 판정) 프레이밍(framing, 틀짓기)

## 새 규칙(New Rule, 새 규칙)

`Tier A(티어 A)`와 `Tier B(티어 B)`는 둘 다 완전히 탐색할 수 있다.

단계 이름(stage name, 단계 이름)은 현재 질문(current question, 현재 질문)을 설명한다. 미래의 모든 알파(alpha, 알파) 작업을 예약하지 않는다.
